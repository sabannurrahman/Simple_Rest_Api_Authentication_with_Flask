1. keputusan desain
- Kerangka Kerja: Flask dipilih karena kesederhanaan dan fleksibilitasnya.
- Database: SQLAlchemy digunakan untuk memetakan objek Python ke basis data.
- Keamanan: JSON Web Token (JWT) digunakan untuk mengelola otentikasi pengguna.
- Penanganan Permintaan: Dekorator seperti butuh_token digunakan untuk menambahkan keamanan ke endpoint-endpoint tertentu.
- Keamanan Tambahan: Praktik keamanan tambahan seperti penggunaan bcrypt untuk hashing password diimplementasikan.
Dokumentasi: Dokumentasi proyek menggunakan format Markdown untuk kemudahan penyajian informasi.
